# 功能需求补充：优先级排序 + 批量导入 + 账号转移

**来源**：用户直接提问（2026-06-12）
**上下文**：多邮箱聚合系统，用户反馈“很多邮箱，重要邮件埋了”、“导入麻烦”、“账号能不能给别人”。

## 总体原则
- 尊重之前调研的架构选项（本地优先桌面 like Pebble / Headless like EmailEngine + 自定义前端 / 纯 Web like Cypht or Zero）。
- 隐私优先：凭证加密、本地计算优先、用户完全控制导出内容。
- 可解释 + 用户可控：不要黑盒排序，让用户理解并能训练/覆盖。
- 国内友好：163/QQ 授权码流程必须顺畅。

## 1. 智能优先级排序（让需要的邮件精确排前面）

### 用户故事
- 作为重度多邮箱用户，我希望打开统一收件箱时，最上面 20 封就是我今天真正需要处理的（老板邮件、客户 deadline、待回复线程），而不是各种订阅和通知。
- 我希望看到“为什么这封排这么高”的解释，并能快速纠正（“这不是重要的”按钮）。

### 功能需求
- **多层混合排序**（确定性规则 > 启发式信号 > 可选 AI）
  - 规则层：用户可创建规则（发件人白名单/VIP、关键词、是否含附件、线程参与、账号权重等）。支持优先级（高/中/低）+ 动作（置顶、标星、移到 Focus 区、静音）。
  - 信号层：发件人历史互动、回复延迟、语义紧急度、跨账号统一评分 + 账号基础权重。
  - AI 层（可选）：本地或云端分类（高/中/低 + 理由），用用户实际操作作为反馈训练。
- **视图**：
  - 统一收件箱默认 “Focus 模式”：顶部重要分区（类似 Gmail Important & Unread），下面是时间流。
  - 支持切换“纯时间排序”。
  - Kanban 看板（Todo/Waiting/Done）作为第二优先级管理层（参考 Pebble）。
- **反馈循环**：星标、归档、回复、拖到看板、点击“不重要” 都会实时影响该发件人/类似邮件的未来分数。
- **可解释性**：每封重要邮件显示简短理由 tooltip。
- **多账号处理**：全局分数 + 每账号颜色徽章 + 允许给“噪声账号”设置低权重或默认过滤。

### 技术建议（参考之前项目）
- 本地优先（Pebble 路线）：在 pebble-rules + 自定义评分 crate 里实现，数据全本地。
- Headless（EmailEngine）：在你的应用层拉取后打分排序，最灵活，可轻松加 AI。
- 规则存储：统一 schema，支持导出（见转移部分）。
- 起步 MVP：规则 + 基础信号（VIP 列表 + 关键词 + 历史回复率），无需 AI。

**成功指标**：用户能通过反馈在 1 周内让 80%+ 高价值邮件排进前 30 封。

## 2. 一键导入多个邮箱账号

### 用户故事
- 我有 12 个邮箱，想一次导入，而不是点 12 次“添加账号”。
- 导入后自动开始同步，并显示每个账号的进度。

### 功能需求
- **主要入口**：CSV / JSON 批量上传 + 预览编辑表格。
- 标准模板字段（见详细文档）。
- 智能补全：识别 163/QQ/Gmail/Outlook 后自动填服务器、端口、安全设置。
- 授权流程：
  - OAuth 账号：引导用户完成授权（可队列处理）。
  - 授权码账号（国内）：支持在 CSV 里带 secret，或导入后批量输入。
- 验证与安全：
  - 上传后 dry-run 测试连接。
  - secret 立即加密存储。
  - 支持“导入设置但不带凭证”（接收方/新设备常用）。
- 额外方式：从备份文件导入（与转移功能复用）、粘贴多行快速添加。
- 进度：导入后进入“同步仪表盘”，每个账号独立进度 + 错误详情。

### 技术建议
- EmailEngine 后端：直接循环调用账号注册 API。
- 本地桌面：批量验证 + 加密存入 SQLite。
- 提供官方 CSV 模板下载 + “如何从 Thunderbird/Outlook 导出账号列表”指南。

## 3. 账号转移 / 分享给别人

### 用户故事
- 我要把我的聚合配置（包括规则、看板、账号列表）给同事/家人/新电脑用。
- 能不能直接把“这个账号的访问权”给别人，而不用分享密码？
- 交接后我还能控制撤销吗？

### 功能需求（按推荐顺序）
1. **配置导出/导入（最重要、最安全）**
   - 导出加密包：账号设置 + 规则 + 标签 + Kanban + saved searches。
   - 凭证：可选包含（加密）、默认不包含（推荐）。
   - 接收方导入后，系统自动提示“以下账号需要重新授权/输入新授权码”。
   - 支持 WebDAV 自动备份（像 Pebble）。

2. **委托/共享访问（Web 多用户场景）**
   - 账号所有者可以把某个账号的权限授予其他用户（只读 / 可回复 / 完全）。
   - 接收方在自己的统一收件箱里看到共享账号，操作时系统使用委托身份。
   - 支持撤销权限。

3. **内容迁移辅助**
   - 提供跨账号移动/复制邮件功能（IMAP 层）。
   - 一键“归档这个账号的旧邮件到另一个账号”。

4. **彻底交接**
   - 导出后一键“清除本地所有该账号的缓存凭证”。
   - 指导用户在源邮箱提供商端撤销旧授权。

### 安全要求（必须写进需求）
- 永远不以明文形式导出/分享授权码或 refresh token。
- 导出文件必须强加密 + 密码提示。
- 导入时明确提示“只有授权接收方才应导入此文件”。
- 审计日志（谁导出了/导入了哪些账号）。

## 推荐在计划中的优先级与阶段

**MVP（必须）**：
- CSV 批量导入（含国内模板 + 智能补全）。
- 基础规则引擎 + 简单信号评分 + Focus 视图（解决“重要邮件排前面”）。
- 加密配置导出/导入（支持转移和多设备）。

**Phase 2**：
- 更强的信号 + 用户反馈循环 + 可解释性。
- Kanban 集成。
- 共享/委托 ACL（如果做 Web 版）。

**Phase 3**：
- AI 增强（本地 LLM 或 API）。
- 高级批量（从现有客户端一键导入）。
- 内容迁移向导。

这些功能和之前调研的 Pebble（规则+备份）、EmailEngine（易于批量 API）、Inbox Zero（AI rules）高度契合。

建议在计划里明确写出“多邮箱噪音处理”和“账号生命周期管理”作为核心用户价值，而不只是“能连多个账号”。
