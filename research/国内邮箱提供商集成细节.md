# 国内邮箱提供商集成细节调研（2026 现状）

**重点**：163/QQ/126/企业邮/企业微信邮箱等对中国用户至关重要的账号。聚合工具必须良好支持，否则无用。

## 1. 通用模式（所有主要国内免费/个人邮箱）
- **不再直接用登录密码**：必须在网页版开启“客户端协议”或“IMAP/SMTP 服务”，系统会生成 **16 位授权码（授权密码）**。
  - 使用授权码代替密码登录第三方客户端/IMAP。
  - 授权码可随时在网页“第三方登录管理”或类似入口新增/管理。
- **收取范围限制**：默认很多只收取最近 30 天邮件。必须在设置中找到“收取全部邮件”或类似选项并开启，否则历史邮件拿不到。
- **SSL 推荐**：IMAP 993、SMTP 465/994/587（STARTTLS 或 SSL）。
- **OAuth 支持**：消费者个人邮箱（163/QQ 等）**基本无现代 OAuth 支持**（或极有限）。主要依赖授权码。企业版有部分 OAuth/扫码/企业微信集成。
- **开启流程通用**：
  1. 网页登录邮箱。
  2. 设置 → POP3/SMTP/IMAP（或账户 → POP3/IMAP/SMTP）。
  3. 开启服务 → 短信/密保验证 → 生成授权码。
  4. 客户端用邮箱地址 + 授权码登录。

## 2. 具体提供商详情

### 网易 163 / 126 / yeah.net 等
- **网页开启**：https://email.163.com/ → 设置 → POP/SMTP/IMAP → 开启 IMAP/SMTP 服务。
- **授权码**：在“邮箱大师”或网页“第三方登录管理 → 通用授权码”新增。
- **服务器（推荐 SSL）**：
  - IMAP: imap.163.com:993 (SSL)
  - SMTP: smtp.163.com:465 (SSL) 或 994
  - POP3: pop.163.com:995 (SSL)
- **企业版 (qiye.163.com)**：
  - 服务器因地区不同（华南/华北等）：imaphz.qiye.163.com / smtphz.qiye.163.com 等（993/994 SSL）。
  - 支持专有协议（降低被盗风险）。
  - 有企业微信对接方案（OAUTH 扫码、消息推送、自建应用、通讯录同步）。
  - 客户端专属协议选项。
- **免费企业邮 (ym.163.com)**：支持 POP3/SMTP，服务器 pop.ym.163.com / smtp.ym.163.com。
- **限制**：部分账号需“收取全部邮件”设置；企业版有更多合规/审计功能。

### QQ 邮箱 / 腾讯企业邮箱
- **开启**：mail.qq.com → 设置 → 账户 → POP3/IMAP/SMTP/Exchange 服务 → 开启 → 短信/密保验证生成授权码。
- **服务器**：
  - IMAP: imap.qq.com:993 (SSL)
  - SMTP: smtp.qq.com:465 (SSL) 或 587
- **腾讯企业邮箱**：类似流程，服务器根据配置（常 smtp.exmail.qq.com 或类似）。支持与企业微信深度集成（升级企业微版、通讯录 API 注意 biz_mail 字段等）。
- **企业微信邮箱对接**：常见场景。支持扫码登录、中间件、消息推送、查看/发送邮件等。网易/腾讯均有对接文档。

### 其他国内常见
- **阿里/钉钉邮箱**：部分支持，类似授权码或企业集成。
- **新浪等**：较少用，类似 IMAP 配置。
- **自建/其他企业邮**：标准 IMAP/SMTP + 可能支持 OAuth2（视服务商）。

**重要注意**：
- 不同邮箱“收取全部邮件”位置表述略不同（邮箱大师 vs 网页）。
- 授权码是**一次性或可管理**的，泄露可单独撤销，不影响网页密码。
- 部分老账号或安全策略严格的可能额外限制第三方访问。
- 企业场景常结合企业微信/钉钉做统一身份 + 邮件提醒。

## 3. 聚合工具集成建议
- **账号添加流程**：引导用户去网页开启 → 复制授权码 → 在你的 App 输入邮箱 + 授权码（不要存明文，立即加密或用 OAuth 代理如果可能）。
- **检测与友好提示**：连接失败时提示“请确认已开启 IMAP 并使用授权码而非登录密码”，并给出图文教程链接（163/QQ 官方帮助）。
- **多账号测试**：强烈建议实际用 163 + QQ + Gmail + Outlook 测试初始同步 + 增量 + 附件 + 搜索。
- **企业场景**：额外支持企业微信扫码/OAuth 流程（参考网易企业微信对接 PDF）。
- **历史邮件**：强调“开启收取全部邮件”并给进度提示（大邮箱初始同步慢）。
- **限额**：国内提供商对第三方客户端有速率/并发限制，做好退避、重试、节流。

## 4. OAuth / 现代认证现状（2026）
- 个人 163/QQ：仍主要授权码。
- 企业版：支持部分 OAUTH 扫码、与企业微信/钉钉集成。
- 网易有旧的 OAuth 文档（Request Token 流程），但现代 Gmail/MS Graph 风格的 client credentials / PKCE 支持有限。
- **结论**：聚合系统必须优雅处理“授权码”登录路径，不能只做 OAuth。IMAP 仍是国内主力。

**对计划的影响**：
- MVP 必须包含清晰的“国内邮箱添加向导”。
- 后端同步层需 robust 处理授权码失效（用户重新授权流程）。
- 如果用 EmailEngine，它应能透传 IMAP 凭据（授权码即密码）。
- 本地优先方案（Pebble 风格）对凭据加密存储特别友好。
