# Email Hub 前端 UI 计划

> 标注：前端 UI。本文只沉淀界面方向、页面拆解和验收标准，不替代后端实施计划。

## 当前基线

前端以浅色三栏工作台为基线，但信息架构对齐成熟邮箱客户端：

- 左侧为全局功能栏，顺序固定为：邮箱、添加邮箱、搜索、Hermes、配置域名、设置。
- 搜索入口在顶部搜索框；邮箱工作台里不重复放左侧搜索工具。
- 第二栏为 Outlook-like 邮箱目录：收件箱、草稿、已发送、已删除、垃圾邮件、归档、所有邮件、标记跟进/提醒状态、附件。
- 不保留“今日优先”或 Smart Inbox 作为固定邮箱文件夹。智能排序可以存在于列表排序或筛选中，但不能替代用户熟悉的邮箱目录。
- 不做独立待办页。跟进事项以 Outlook-style 旗标、标记跟进、提醒、待回复状态附着在邮件上。
- 中间为邮件列表，支持全部、未读、已标记、待回复等顶部筛选。
- 右侧为邮件阅读区：工具栏、邮件正文、附件、回复/转发编辑模块。
- 底部 Hermes 输入保留为邮箱命令入口，但未配置时只做轻提示并引导到 Hermes 页面。

## 页面清单

P0 必须先完整设计：

- 邮箱主工作台
- 添加邮箱
- Gmail / Outlook OAuth 队列
- 163 / QQ 授权码引导
- Proton Bridge 连接
- iCloud 邮箱：Apple app-specific password 引导
- 个人域名邮箱：通用 IMAP/SMTP 手动配置
- 同步中心（内部授权/重连页，不放主导航）
- 搜索页
- 写邮件 / 回复交互
- 批量导入
- 账号转移
- 配置域名
- 设置页
- 移动端浏览器响应式完整体验

## Hermes 前端定位

Hermes 是唯一 AI 入口，但普通用户只配置服务连接：

- Hermes 页面只显示：用户可编辑的助手名称、LLM 提供商、API Key、测试连接。
- 默认提供商覆盖主流服务商、NVIDIA、OpenAI-compatible 自定义 provider。
- 技能、规则、memory、audit log、资源预算、系统提示词等均为系统内置能力，不能显示在普通 Hermes 页面。
- 底部 Hermes 输入框用于搜索邮件、写回复、整理收件箱。
- 写操作必须进入预览，不直接发送。
- 未配置 Hermes 连接时，底部 Hermes 可以灰显并引导到 Hermes 页面，不要求用户配置 env。

## Compose 与回复

- 新写邮件对齐 Outlook：打开浮动写信窗口，不挤压三栏邮箱。
- 浮动窗口要有明确的打开/关闭动态效果，方向是苹果设备式缩放过渡，避免生硬出现。
- 回复和转发对齐 Outlook：可以在最右侧阅读区内直接出现可编辑模块。
- 用户可以在浮动窗口和阅读区回复之间形成自然流转，但三栏布局不应被写信面板向下挤压。

## 配置域名

- 左侧入口名称为“配置域名”。
- 第一层页面保持干净：添加域名、DNS 状态、别名/转发地址、验证结果。
- 支持手动 DNS 记录复制和自动检测。
- 支持 Cloudflare 自动配置，放在清晰的自动配置流程中，说明 API Token 权限边界。
- Cloudflare 只是 DNS 托管商接入，不等同于邮箱服务商；除非明确使用 Cloudflare Email Routing，否则不要混淆两者。

## 滚动规则

- 桌面端：整屏铺满，左侧全局栏固定。
- 邮箱主工作台：第二栏目录、邮件列表、阅读区各自可滚动。
- 非邮箱页面：主区域整体可滚动。
- 底部 Hermes 不应遮挡关键按钮，阅读区和列表底部要留出安全间距。
- 桌面和移动浏览器都要完整可用，不能把移动端当成独立 App 简化掉。
- 1440 桌面和 390 移动端不能出现横向溢出、文字重叠。
- 桌面功能栏和内容栏需要自动适应屏幕大小，后续支持用户拖拽调整关键列宽。

## Smart Inbox 展示规则

排序不能只显示 AI 结论，必须显示原因：

- 直接发给你
- 你常回复此发件人
- VIP / 客户规则
- 今天 17:00 截止
- Hermes 识别为需要回复
- 来自项目标签
- newsletter / bulk sender 扣分

智能能力应表现为排序、筛选、原因提示、旗标建议或 Hermes 预览，不应把邮箱目录变成 AI 分类中心。

## 验收标准

- 左侧全局栏顺序为邮箱、添加邮箱、搜索、Hermes、配置域名、设置。
- 搜索以顶部搜索框为主。
- 第二栏完整包含 Outlook-like 邮箱目录，不保留今日优先/Smart Inbox 固定文件夹。
- 待办不作为独立页面；标记跟进、提醒和待回复是邮件状态。
- Hermes 页面只显示名称、provider、key、测试连接。
- 配置域名页面支持手动 DNS、检测、Cloudflare 自动配置。
- 底部 Hermes 是长条输入，不是短小自动隐藏形态。
- 邮件列表点击后阅读区同步切换。
- 邮件阅读区支持右侧内联回复/转发编辑。
- 新写邮件以浮动窗口打开，不挤压三栏。
- 页面能铺满桌面宽度。
- 主要内容区可以上下滚动。
- 颜色符合浅薄荷灰、低饱和珊瑚红、中性灰白。
- 通过 `npm test` 和 `npm run build`。
